# UI 原型说明

## 视觉方向

Step 0 已锁定为浅色蓝白政企 SaaS 风格，面向高信息密度 B 端后台，不做营销页式首页，也不做暗色霓虹风格。

- 蓝白主色，辅以青绿、橙色和红色作为状态色。
- 顶部全局导航 + 左侧模块菜单。
- 页面信息密度高，但区块边界清晰。
- 表格、筛选、时间线、统计卡片和流程画布是核心组件。
- 避免营销式大 hero，首页直接是总览工作台。

## Step 0 已确认决策

- 模板复制变量统一展示为 `{{ payload.title }}`，内部路径仍保持 `payload.title`。
- 路由策略按顺序执行，第一条命中即停止继续匹配，并显示累计命中次数。
- `无鉴权` 必须使用黄色风险标签，不与安全鉴权方式同权重展示。
- 页面默认 5 秒轮询，并保留手动刷新入口，不使用 SSE。
- 队列监控是独立页面，不挂在日志或总览的二级面板里。

## 当前管理台 UI 基线

截至 2026-05-21，原型说明以当前实现为准：

- 总览顶部指标固定为总接收量、总发送量、成功发送量、失败发送量、成功率、平均 OPS；不再展示“活跃平台数”。空值和异常值必须兜底为安全展示，不出现 `NaN`。
- 指标卡可使用 Retina 徽章、SVG 走势暗纹、故障尖峰线、成功趋势线和健康 Donut 等微型图表，但必须服务于指标语义。
- 推送渠道使用品牌化类型卡片和 Bento Grid 选择器，图标、品牌色和启用实例数要形成统一视觉语言。
- AppShell 使用受控子 Tab 与 Workspace Keep-Alive，切换已打开标签页保留状态、滚动、分页和输入，不重复刷新。
- 表格长文本字段使用截断与 Tooltip；队列监控使用外层页面滚动和磨砂玻璃 sticky 表头。
- 人员详情中的平台身份验证态低噪声展示：已验证为淡灰胶囊，未验证不展示空标签。

## 功能模块原型

### 原型资产索引

当前未生成图片资产，因此将 9 张模块的 gpt-image2 提示词、构图、关键控件和验收点整理在 [`generated/README.md`](generated/README.md)。后续如需直接出图，可按该文件逐条生成。

### 通用列表交互

所有带列表的模块统一采用“查询栏 + 分页表格 + 新增按钮 + 弹窗/抽屉新增编辑”的交互结构。字段中文名、枚举中文名和状态标签以 `list-field-status-spec.md` 为准，前端不得直接展示英文枚举值。

### 总览

目标：展示 24h 趋势、QPS、成功率、失败率、队列积压、平台排行和错误排行。

布局：

- 顶部：关键指标卡。
- 中部：24h 趋势图和 QPS 图。
- 右侧：失败原因排行、最近异常。
- 下部：平台维度发送量和成功率。

### 来源接入

目标：配置下级系统、鉴权、IP 白名单、入站去重、入站限流和消息免打扰；入站格式固定为标准 JSON，不在页面配置。

布局：

- 上方查询栏：来源名称、编码、状态、鉴权方式。
- 主体表格：来源列表。
- 详情抽屉：基础信息、鉴权、IP 白名单、入站去重、入站限流和消息免打扰；最近 payload 只在来源列表“查看”中展示。
- 鉴权表单默认选择 Token，并展示 `Authorization: Bearer <source_token>` 使用说明。
- HMAC 是可选开关，开启后支持随机生成共享密钥。
- 支持 Token + HMAC 双校验模式，前端文案为“Token + HMAC 双校验”。
- 选择无鉴权时显示风险提示，并建议填写 IP 白名单。
- 消息免打扰默认关闭，开启后展示最多 5 个时间段，支持 `22:00 ~ 08:00` 这类跨天设置。
- 来源列表的“鉴权方式”列中，“无鉴权”必须显示黄色风险标签。

### 推送渠道

目标：配置飞书、钉钉、企微、邮箱、短信、政务云、Webhook、自建服务、个人推送和自定义 Token 推送渠道。

布局：

- 左侧品牌化推送渠道类型筛选，展示统一图标底座、品牌色和启用实例数。
- 中间推送渠道实例列表。
- 右侧能力摘要：消息类型、接收人字段、Token 策略、请求结构。
- 高级配置用分段表单：Token 获取、发送请求、接收人映射、主动限流、并发上限、超时重试、死信策略、测试。
- 新增渠道时使用可搜索、可分组的 Bento Grid 品牌卡片选择器。
- 渠道测试区提供 AccessToken 强制刷新按钮；刷新由后端执行，前端只展示刷新状态和时间。

### 路由编排

目标：用无限画布配置来源、条件和多个结束节点，同时保留传统表格。

画布模式：

- 左侧节点库：条件、模板、接收人、平台动作、去重、错误处理。
- 中间无限画布：单个来源起点，多条件分支，多结束节点。
- 右侧属性面板：配置当前节点。
- 底部：校验结果、模拟运行、发布版本、当前执行版本。

传统模式：

- 一行一条规则。
- 支持拖拽排序、上移、下移，按顺序执行。
- 列包括顺序、条件、模板、接收人策略、目标平台、去重、命中次数、启停。
- 命中次数类似防火墙策略命中次数，新建从 0 开始，不因排序、编辑或发布新版本清零，最高显示 99999。
- 第一版执行语义为“第一条命中即发送并停止继续匹配”。

### 模板中心

目标：模板编辑、字段复制、实时预览和保存前校验。

布局：

- 左侧：来源选择、最近 payload 选择、自动解析按钮、字段两列表。
- 字段表第一列：可复制变量，例如 `{{ payload.title }}`。
- 字段表第二列：当前样例值。
- 右上：模板编辑器和消息体编辑器。
- 右下：实时预览、schema 校验、错误列表。
- 保存按钮在校验失败时禁用。

### 组织人员

目标：组织树和人员管理，适配不同推送渠道身份字段。

布局：

- 左侧可收起组织树。
- 中间人员表。
- 右侧人员详情：基础信息、平台身份字段、所属组织。
- 支持新增、编辑、删除、移动组织、导入。

### 匹配组

目标：维护条件判断复用组。

布局：

- 左侧组列表。
- 中间组内值表。
- 右侧引用情况和测试匹配。

### 消息日志

目标：入站和出站统一查询。

布局：

- 上方查询区：时间、来源、平台、状态、trace、错误码、关键字。
- 主体分页表格：一条入站主记录。
- 详情页：入站 payload、命中路由、出站请求/响应、时间线。
- 无出站记录时，出站字段显示 `-`。

### 队列监控

目标：单独展示队列积压、worker 处理能力、渠道限流和慢规则，帮助判断系统是否被某个来源、路由或推送渠道拖慢。

布局：

- 顶部：`route_plan` 待处理、`send_message` 待处理、最老 job 等待时间、事务总平均耗时、成功率。
- 中部左侧：队列积压趋势、planning 平均耗时 / P95、sending 平均耗时 / P95。
- 中部右侧：平台实例健康表，列包括待发送、失败率、限流次数、重试次数、死信数量。
- 下部：慢规则列表，列包括来源、路由组、规则、命中次数、平均耗时、P95。
- 右上角：手动刷新；页面默认 5 秒轮询。
