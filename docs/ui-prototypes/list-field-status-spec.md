# 列表、字段和状态中文映射规范

## 通用列表规范

所有管理对象和运行记录默认使用统一列表形态：

- 顶部查询栏：按模块提供关键筛选项，支持重置和查询。
- 查询按钮必须按当前条件重新请求后端数据；重置按钮必须清空条件并重新加载默认列表。
- 右上操作区：主按钮“新增”，必要时提供导入、刷新、批量操作。
- 主体分页表格：服务端分页、服务端排序，默认每页 20 条。
- 行操作：查看、编辑、删除、启停、测试、重试等；第一版不做 RBAC，管理员单账户全部可见。
- 新增：点击“新增”按钮后打开弹窗或右侧抽屉，不跳转到单独页面。
- 编辑：复杂表单优先右侧抽屉；短表单可用弹窗。
- 详情：日志、路由、平台等复杂对象使用详情页或详情抽屉。
- 前端表格、表单、筛选项、状态标签不得直接展示英文枚举。
- 列表容器高度按页面内容统一拉齐，底部保留和“来源接入”一致的空隙；表格内容超过限定高度时使用内部滚动。
- 空状态应在表格可视区居中展示，不额外显示割裂的底部分隔线。
- 浏览器验证时，布局数据以固定 16:9 或 1080p viewport 为准；纯视觉截图可额外最大化浏览器窗口。
- 长文本列必须使用行内块级 `ellipsis` 截断并配合 Tooltip 展示完整值，至少覆盖规则名称、条件、发送动作组、模板内容字段、操作审计资源名称和 Trace ID。
- 全局表格表头允许使用 sticky 与 `backdrop-filter: blur(8px)` 磨砂玻璃效果；队列监控类长页面优先由外层页面原生滚动承载，不在内部硬塞固定高度。

适用对象：

- 来源接入列表。
- 推送渠道列表。
- 模板列表。
- 路由组列表。
- 子路由/规则列表。
- 组织列表和人员列表。
- 匹配组和匹配项列表。
- 接收人组列表。
- 消息日志列表。
- 出站尝试列表。
- 操作审计列表。
- 队列监控列表。
- 死信任务列表。

## 主菜单和聚合页

当前主菜单固定为：

- 总览。
- 来源接入。
- 推送渠道。
- 消息模板。
- 路由策略。
- 日志监控。
- 组织人员。
- 系统设置。

聚合页结构：

- 路由策略：路由组、匹配组。
- 日志监控：消息日志、队列监控、操作审计。
- 组织人员：人员管理、接收人组；人员管理内嵌组织树和组织节点维护。
- 系统设置：系统参数。

旧入口 `logs`、`queue`、`audit` 统一映射到“日志监控”，`matchGroups` 映射到“路由策略”。右上角实时通知只提供“查看日志监控”入口，不创建额外日志页面。

AppShell 需要顶层受控聚合页子 Tab：

- 从旧版子页面链接进入时，主框架应跳转到对应聚合页，并同步高亮内部 Tabs。
- Workspace 多标签页切换采用页面级 Keep-Alive。已打开页面通过 `display: none/block` 保持挂载，保留表单输入、滚动位置、分页和局部状态。
- 切换 Workspace 标签页不得造成重复网络刷新；只有点击左侧主导航或显式刷新时才重新拉取页面数据。
- 子 Tabpane 的布局样式必须过滤 `.ant-tabs-tabpane-hidden`，防止隐藏面板参与弹性高度计算造成垂直堆叠。

## 组织人员页面规范

- 组织人员拆分为“人员管理 / 接收人组”两个子页面。
- 人员管理左侧展示组织树，右侧展示人员列表、人员筛选和人员详情；组织节点的新增、编辑、删除在该页内完成。
- 组织树节点操作用于新增下级组织，并预置上级组织。
- 新增组织主按钮样式与新增人员保持一致。
- 人员管理维护手机号、邮箱和各推送渠道身份字段；手机号/邮箱作为 email/SMS 接收人解析的最终兜底，渠道实例级和 provider 默认身份优先。
- 人员详情 Drawer 的平台身份验证态使用低噪声样式：已验证显示淡灰 `default` 胶囊标签；未验证状态直接隐藏，不显示空边框胶囊。
- 接收人组归属组织人员页；路由策略页只引用接收人组，不重复维护。

## 总览和趋势图规范

- 时间筛选表示“最近 N 分钟 / 最近 N 小时 / 最近 7 天”的滚动窗口。
- X 轴必须跟随后端返回的 `bucket_start` 变化，不固定显示 0-24。
- 分钟和小时窗口使用 `HH:mm` 标签，7 天窗口使用 `MM/DD` 标签。
- 0 值数据点不显示数值标签，避免和 Y 轴重叠。
- 没有数据时展示基线和空状态，不使用前端 mock 统计。
- 平台发送量、成功率、失败排行、右上角通知数字都从后端统计或日志接口获取。
- 顶部指标卡顺序固定为：总接收量、总发送量、成功发送量、失败发送量、成功率、平均 OPS；不再展示“活跃平台数”。
- 所有数字、百分比和时长格式化函数必须对 `null`、`undefined`、`NaN` 和异常类型做防御，统一兜底为 `0`、`0.00%`、`0 ms` 或同类安全展示。
- 指标卡微型图表必须语义化：接收/发送使用趋势线，失败使用尖峰或异常线，健康度使用 Donut 或成功率表达。动效应轻量，只在 hover 等微交互中触发。

## 账户菜单规范

- 管理员头像点击后展示账户菜单。
- 账户信息显示登录用户名和账户别名；账户别名可编辑，用户名不可编辑。
- 修改密码必须输入旧密码、新密码、确认新密码；新密码不少于 10 位，两次输入必须一致。
- 密码规则不满足时前端直接展示中文校验，不只透传后端错误码。
- 退出登录必须弹出二次确认。

## 来源接入字段

| 字段 | 中文 | 展示要求 |
|---|---|---|
| `source_code` / `code` | 来源编码 | 等宽文本；来源创建后不可修改 |
| `name` | 来源名称 | 普通文本 |
| `auth_mode` | 鉴权方式 | 标签 |
| `ip_allowlist` | IP 白名单 | CIDR、单 IP、IP 段列表 |
| `inbound_dedupe_enabled` | 入站去重 | 开关/标签 |
| `rate_limit_config` | 入站限流 | 摘要 |
| `do_not_disturb_config` | 消息免打扰 | 编辑来源中配置；最多 5 个时间段，支持跨天 |
| `latest_payload_sample` | 最近 Payload | 来源列表操作列点击“查看”后展示 |

### 来源鉴权方式

| 值 | 中文 | 标签颜色 | 说明 |
|---|---|---|---|
| `token` | Token | 蓝色 | 默认值，生产环境支持 |
| `hmac` | HMAC | 绿色 | 使用共享密钥签名 |
| `token_and_hmac` | Token + HMAC 双校验 | 紫色 | Token 和 HMAC 必须同时通过 |
| `none` | 无鉴权 | 黄色 | 风险状态，必须提示建议配置 IP 白名单 |

### 推送渠道类型

| 值 | 中文 |
|---|---|
| `wecom` | 企业微信 |
| `feishu` | 飞书 |
| `dingtalk` | 钉钉 |
| `email` | 邮箱 |
| `sms` | 短信 |
| `self` | MVP-PUSH |
| `webhook` | 通用 Webhook |

推送渠道类型侧栏展示要求：

- 类型按用途分组，组标题展示为“X 个”，不使用“X 类”。
- “全部渠道”和下方类型项之间保留清晰间距。
- 类型项使用微型品牌卡片，展示统一图标底座、品牌色和当前启用实例数；允许横向滑动和轻微弹性动效。
- 类型项宽度按最长中文标签适配，不强制铺满侧栏。
- 侧栏默认宽度以当前文字长度为准，不和组织树使用同一宽度。
- 创建渠道时，推送渠道类型选择器优先使用可搜索、可分组的三列 Bento Grid 品牌卡片；选中态使用清晰 check 徽标，不回退为纯文本 `<Select>`。
