# 2026-06-08 管理台与性能链路会话记录

本记录用于接续后续开发，概括本轮从八个菜单审查到性能链路改造期间已经落地的主要变更和仍需注意的边界。

## 管理台菜单修复

- 总览：补齐多序列趋势、平均耗时、QPS/耗时趋势和推送数据表展示；平台类型使用图标加文字，渠道名加粗，数字列使用等宽展示。
- 来源接入：优化来源列表文案和策略展示；入站测试会进入日志；来源最近 Payload 展示去掉多余灰框。
- 推送渠道：保留并发配置，列表展示并发列；梳理死信处理思路；去掉不准确的默认描述和部分误导性文案。
- 消息模板：模板版本固定语义落地；版本历史展示模板内容、接收效果、最近 Payload 和出站 JSON；变量展示改为轻量列表。
- 路由策略：画布新增节点联动规则草稿；规则列表语义向当前执行版本靠拢；匹配组查看态禁止编辑并展示引用关系。
- 日志监控：菜单名统一为“日志监控”；消息日志详情时间格式统一；“入站主记录”文案改为“日志列表”；出站时间改为上级调用结束时间，不再使用进入出站队列时间。
- 组织人员：手机号/邮箱作为 common 身份兜底；接收人组成员引用校验；平台身份字段展示收敛为单行主文本。
- 系统设置：性能测试页改为分档压测报告，展示推荐并发、出站 QPS、出站 P99、资源诊断、吞吐和延迟曲线；支持取消和二次确认。

## 性能链路与队列

- 引入 NATS JetStream 链路，将入站、规划和投递事件从 PostgreSQL 表队列中拆出，降低核心链路写库阻塞。
- RoutePlanEvent 携带 payload/source/trace 信息，规划阶段可基于事件做路由判断和发送发布，日志由旁路异步补写。
- 最新 Payload 样例改为低频异步刷新思路，并讨论多实例下可用 NATS KV 承载最新状态。
- HMAC nonce 和入站去重已按 NATS KV/CAS 方向收敛，降低 PostgreSQL 锁竞争。
- 路由配置仍以 PostgreSQL 为准；发布、启用、修改后通过 NATS subject 广播 `route.changed`，实例收到后刷新对应内存缓存，并保留定时全量刷新兜底。
- 性能统计口径拆分为接收、出站和完整链路；出站 QPS/P99 以“入站到请求发出”为主，不再把结果落库混入出站口径。

## 通知与刷新

- 右上角通知使用 `/monitor/notifications/stream` SSE，而不是前端定时轮询一批列表接口。
- SSE 推送间隔读取系统参数 `console.polling_interval_seconds`，默认 5 秒；修改参数后新连接和后续轮次会按新值执行。
- 页面列表和详情不因 SSE 自动刷新；当前策略是首次进入、手动刷新、导航切换和局部操作后显式刷新。
- 账户按钮已通过 `Space align="center"`、`.header-actions` 和 `.ant-space-item` 统一垂直居中，按钮自身不再额外下移。

## 安全与公开部署边界

- 当前产品基线仍是单管理员、单租户。数据库表没有 `tenant_id`，不适合直接作为公网多用户 SaaS 开放注册。
- 来源 token、HMAC secret、渠道 `auth_config/token_config`、部分 access token 和日志 payload 当前按第一版基线明文落库或明文展示；密码和后台 session token 使用哈希存储。
- 若要公网多用户多租户，需要新增 tenant/account/membership、所有租户数据表加 `tenant_id`、所有 Repository 强制租户过滤、SSE/NATS/KV/cache key 加租户前缀，并补 RBAC、密钥加密、CSP、登录限流、审计脱敏和跨租户 IDOR 测试。

## 当前验证习惯

- 前端 UI 改动至少跑对应 Vitest 和 `npm run build`。
- 后端 handler/service 改动至少跑目标包 `go test`；涉及公共链路时跑 `go test ./... -count=1`。
- 完成前继续使用 `git diff --check`，避免格式和尾随空白问题。
