# 2026-05-21 管理台 UI 与 AccessToken 刷新改造记录

本文记录 2026-05-21 管理台视觉、交互、布局和渠道 token 刷新能力的阶段性基线。后续实现、回归和文档更新应以当前源码为准。

## 总览大盘

- 顶部核心指标顺序调整为：总接收量、总发送量、成功发送量、失败发送量、成功率、平均 OPS。
- “活跃平台数”不再作为核心指标；“总接收量”来自后端统计，用于表达网关实际入站吞吐。
- 数值、百分比和耗时格式化函数需要防御 `null`、`undefined`、`NaN` 和异常类型，空数据时展示 `0`、`0.00%`、`0 ms` 或等价安全默认值。
- 指标卡使用语义化微型图表：接收/发送趋势线、失败尖峰线、成功趋势线、健康 Donut 等；动效只作为 hover 微交互，不影响阅读。

## 推送渠道视觉与选择器

- 推送渠道类型使用统一品牌视觉资产，包含图标底座、品牌色和类型说明。
- 左侧类型导航升级为微型品牌卡片，并展示启用实例数。
- 新增渠道类型选择器使用可搜索、可分组的 Bento Grid 品牌卡片，不再以纯文本下拉作为主体验。

## AccessToken 强制刷新

- 后端提供 `/api/v1/channels/{id}/refresh-token`，由服务端访问上级 token 接口并同步本地缓存状态。
- 前端渠道测试区域提供紧凑刷新按钮，展示异步加载状态和最近刷新时间。
- `access_token` 不返回前端，不由前端拼接或缓存。

## AppShell、Tabs 和 Keep-Alive

- AppShell 顶层受控聚合页子 Tab，旧子页面链接应能驱动主框架跳转并同步内部 Tabs 高亮。
- Workspace 多标签页采用页面级 Keep-Alive，已打开页面通过 `display: none/block` 保持挂载，保留输入、滚动、分页和局部状态。
- 切换 Workspace 标签页不应造成冗余网络刷新；点击左侧主导航或显式刷新才重新拉取数据。
- 子 Tabpane 布局样式必须过滤 `.ant-tabs-tabpane-hidden`，避免隐藏面板参与弹性高度计算造成垂直堆叠。

## 表格和布局细节

- 规则名称、条件、发送动作组、模板内容字段、操作审计资源名称和 Trace ID 等长文本列使用截断与 Tooltip。
- 队列监控释放内部固定高度约束，滚动交给页面最外层原生滚动条。
- 全局表格表头可使用 sticky 和 `backdrop-filter: blur(8px)` 磨砂玻璃效果。

## 组织人员平台身份

- 人员详情 Drawer 中，平台身份已验证状态使用淡灰 `default` 胶囊标签。
- 未验证状态不展示空胶囊，直接隐藏，降低视觉噪声。
